# PolarFP-VINS 系统架构图绘制 Prompt

> 用于网页 AI 绘图工具生成论文方法架构图 (System Architecture Diagram)，风格参考 IEEE T-RO / ICRA 顶会顶刊论文。

## 总体描述

绘制一张机器人视觉惯性导航系统的端到端方法架构图，简洁、专业、信息密度高。整体布局从左到右分为五大阶段，背景为白色，使用扁平化设计风格。

## 画布设置

- **推荐宽高比**：16:9（例如 1920 × 1080 px）
- 如需适配 IEEE 双栏论文单栏宽度，导出时保证宽度 ≥ 800 px 即可缩放
- 如需适配 IEEE 双栏论文跨栏宽度，导出时保证宽度 ≥ 1700 px 即可缩放

## 布局与内容

### 阶段一：传感器输入（最左侧，窄列）

- 顶部：矩形框标注 **"偏振相机"**
  - 向下箭头标注 **"原始偏振图像（微偏振阵列）"**
- 底部：矩形框标注 **"六轴 IMU"**
  - 灰色虚线箭头标注 **"100 Hz IMU 数据"**，直接指向右侧阶段五的后端区域

### 阶段二：偏振通道解码（创新点 a，浅黄色背景框高亮）

从"原始偏振图像"引出，进入处理框：

```
┌──────────────────────────┐
│   斯托克斯矢量计算        │
│   S₀ = ½(I₀+I₄₅+I₉₀+I₁₃₅)│
│   S₁ = I₀ − I₉₀         │
│   S₂ = I₄₅ − I₁₃₅       │
└───────────┬──────────────┘
            ↓
┌──────────────────────────┐
│   DoP 与 AoP 计算         │
│   DoP = √(S₁²+S₂²)/S₀   │
│   AoP = ½·atan2(S₂,S₁)  │
└───────────┬──────────────┘
            ↓
     分为四路输出（4 个并排小方框，水平排列）：

  ┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐
  │  S₀ │  │  P₀ │  │  P₁ │  │  P₂ │
  │强度图像│ │偏振度│ │sinAoP│ │cosAoP│
  └─────┘  └─────┘  └─────┘  └─────┘
```

- S₀ 框用**蓝色**边框，框下方标注 **"暗光下信噪比低"**
- P₀ 框用**橙色**边框，P₁ 框用**绿色**边框，P₂ 框用**红色**边框，各框下方标注 **"暗光下仍保留结构信息"**
- 在 P₀, P₁, P₂ 上方增加处理框（浅黄色底色）：

```
┌───────────────────────┐
│  S₀ 导向滤波           │  ← 核心创新，用浅黄色背景高亮
│  （保边去噪）          │
└───────────────────────┘
```

用三条细线从该框分别连到 P₀, P₁, P₂ 三个通道。

**阶段旁标注（右侧空白处）：**
> 创新点 (a)：暗光特征增强——偏振通道解算 + S₀ 导向滤波预处理

### 阶段三：多通道特征跟踪

四个通道各自进入独立的处理流水线（4 条并行的水平轨道，轨道颜色分别对应蓝/橙/绿/红）：

每条轨道包含两个串联小框：

```
┌──────────────┐    ┌──────────────┐
│ 特征点检测    │ →  │ L-K 光流跟踪  │
│  （GFTT）    │    │              │
└──────────────┘    └──────────────┘
```

在四条轨道上方标注：
> "各通道独立检测与跟踪，分配通道局部 ID"

每条轨道上的特征点/连线用对应通道颜色（蓝/橙/绿/红）表示。

### 阶段四：全局特征池（创新点 b，浅黄色背景框高亮，图中最大最核心模块）

一个较大的矩形区域，内部从上到下分为三个子阶段：

```
┌─────────────────────────────────────────────────────────┐
│  全局特征池 Global Feature Pool                          │
│                                                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │ ① 传播 Propagation                              │   │
│  │  "检查上一帧全局特征的关联通道是否仍被跟踪"       │   │
│  │  示意：上一帧的全局特征点（紫色圆点）              │   │
│  │  → 各通道局部特征（蓝/橙/绿/红小圆点）             │   │
│  │  → 存活通道重新绑定观测                           │   │
│  └────────────────────┬────────────────────────────┘   │
│                       ↓                                │
│  ┌─────────────────────────────────────────────────┐   │
│  │ ② 关联 Association                              │   │
│  │  "空间哈希网格匹配（距离阈值 τ = 4 px）"          │   │
│  │  示意：3×3 空间哈希网格                          │   │
│  │  新局部特征（蓝/橙/绿/红小圆点）                   │   │
│  │  → 按距离合并到已有全局特征（紫色圆点）            │   │
│  │  → 无法关联则创建新全局特征                       │   │
│  └────────────────────┬────────────────────────────┘   │
│                       ↓                                │
│  ┌─────────────────────────────────────────────────┐   │
│  │ ③ 输出 Output                                   │   │
│  │  "通道优先级：S₀ → P₀ → P₁ → P₂"                │   │
│  │  每个全局特征每帧仅输出一个通道的观测              │   │
│  │  → 后端观测向量：                                │   │
│  │    [x_norm, y_norm, 1, u, v, vx, vy]            │   │
│  │  输出特征统一用紫色表示                           │   │
│  └─────────────────────────────────────────────────┘   │
└───────────────────────┬─────────────────────────────────┘
                        ↓
          特征帧 FeatureFrame（全局 ID → 观测向量）
          所有特征点用紫色表示（统一融合特征）
```

**阶段旁标注（右侧空白处）：**
> 创新点 (b)：跨通道全局特征池——特征生命周期不再依赖单一通道

### 阶段五：VIO 后端优化（右侧区域）

```
┌──────────────────────────────────────────┐
│  滑动窗口非线性优化（Ceres Solver）       │
│                                          │
│  输入：                                    │
│  · 全局特征观测（来自前端，紫色）           │
│  · IMU 预积分（来自 IMU）                 │
│                                          │
│  因子图示意：                               │
│                                          │
│    ○──────○──────○──────○                │
│   帧0     帧1    帧2    帧3               │
│    │       │      │      │               │
│    │  IMU  │ IMU  │ IMU  │               │
│    │  因子 │ 因子 │ 因子  │               │
│    │       │      │      │               │
│    ●───────●──────●──────●                │
│    全局特征重投影因子（紫色）               │
│                                          │
│  注：所有视觉特征均为全局融合特征           │
│  （紫色），不再区分 S₀/P₀/P₁/P₂ 通道       │
│                                          │
│  输出：                                    │
│  位姿 P ∈ R³，四元数 Q ∈ SO(3)            │
│  速度 V ∈ R³，IMU 偏差 Ba, Bg            │
└──────────────────────┬───────────────────┘
                       ↓
              ┌─────────────────┐
              │   估计位姿与轨迹  │
              └─────────────────┘
```

- 因子图中的位姿节点 ○ 用灰色圆圈表示
- 视觉特征重投影因子 ● 统一用**紫色**（#9B59B6）表示，标注 **"全局特征重投影误差"**
- IMU 因子连线用灰色，标注 **"IMU 预积分约束"**

**阶段旁标注（右侧空白处）：**
> 创新点 (c)：完整 PolarFP-VINS 系统——极暗光（0.9–4.8 lux）下匹配特征数提升约 6 倍

## 风格要求

- 扁平化设计，无 3D 效果，无阴影渐变
- 模块使用圆角矩形
- 箭头清晰，数据流向从左到右明确
- 所有文字使用中文，sans-serif 字体
- 图表紧凑，不留过多空白
- 五个阶段之间用细竖线或浅色背景分区区分
- 图底部可添加图例（Legend），说明五种颜色的含义：
  - 🔵 S₀（强度通道）
  - 🟠 P₀（DoP 通道）
  - 🟢 P₁（sinAoP 通道）
  - 🔴 P₂（cosAoP 通道）
  - 🟣 统一融合特征（全局特征池输出）
